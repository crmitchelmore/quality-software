# Nielsen's Usability Heuristics

> Ten broad interface principles for finding usability problems quickly, from visibility of system status to error prevention and recovery.

**Discipline:** ux · **Origin:** Jakob Nielsen · *10 Usability Heuristics for User Interface Design* · (1994)

**Also known as:** 10 Usability Heuristics

## Description

Nielsen's usability heuristics are a compact set of design principles intended for heuristic evaluation: expert reviewers inspect an interface and identify likely usability problems without waiting for a full user study. They are deliberately general rather than prescriptive rules, covering system feedback, match with the real world, user control, consistency, error prevention, recognition over recall, flexibility, minimalism, error recovery and help. Their enduring value is speed and shared vocabulary: teams can cheaply catch common usability failures before or alongside empirical testing.

**In practice.** Have several trained reviewers walk through important tasks, record violations of specific heuristics, rate severity, then prioritise fixes that reduce task failure or confusion. Use the heuristics as a screening method, not as a substitute for watching real users.

## Core tenets

- Keep users informed about system status with timely, meaningful feedback.
- Speak the user's language and align the interface with real-world concepts.
- Support user control, freedom, undo and recovery from mistakes.
- Prefer consistency, standards and recognition over forcing users to remember.
- Prevent errors where possible and make unavoidable errors easy to diagnose and fix.

## Key ideas

- **Heuristic evaluation** — A small number of evaluators independently inspect an interface against the heuristics, then consolidate severity-rated findings.
- **Recognition rather than recall** — Interfaces should make options, state and instructions visible so users do not have to remember hidden information.
- **Error prevention and recovery** — Good design both reduces the chance of mistakes and provides clear paths back when they occur.

## Criticisms / limits

- Expert inspection predicts likely issues but cannot prove real users will experience them.
- Findings depend heavily on evaluator skill and domain knowledge.
- The heuristics are broad, so teams may need accessibility and content-specific checks as well.
